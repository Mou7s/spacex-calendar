# SpaceX 发射日历 — 可提升方向


## 🔧 代码 / 架构

| 方向 | 说明 |
|---|---|
| **翻译数据外置** | 目前中英文翻译硬编码在 `app.js` 里（占了很大篇幅），可以拆到独立的 JSON 文件 |

## 🚀 功能

| 方向 | 说明 |
|---|---|
| **发射提醒通知** | 利用 Web Notification API，在发射前 N 小时推送浏览器通知 |
| **时区选择器** | 允许用户手动选择时区，而不只是依赖浏览器默认时区 |
| **发射历史归档** | 当前只展示 upcoming 任务，可以增加一个"已完成"标签页 |
| **日历视图点击日期** | 点击缩略图中的某个日期，自动滚动右侧事件列表到该日期 |
| **分享功能** | 单独任务卡片可以一键分享链接 |

## 📊 性能 / SEO

| 方向 | 说明 |
|---|---|
| **数据缓存策略** | Worker 端可以用 Cloudflare KV 缓存上游数据，减少实时请求 |
| **Open Graph 元标签** | 添加 OG 标签，在社交媒体分享时显示预览卡片 |
| **结构化数据** | 添加 JSON-LD（Schema.org Event），有助于 Google 搜索展示 |

## ✅ 测试

| 方向 | 说明 |
|---|---|
| **前端测试** | 目前只有后端逻辑测试，可以为 `app.js` 的渲染函数加单元测试 |
| **E2E 测试** | 用 Playwright 做端到端测试，验证页面交互流程 |
